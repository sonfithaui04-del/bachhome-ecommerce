1. Kiến Trúc Tổng Quan

Hệ thống áp dụng kiến trúc Microservices với các đặc điểm kỹ thuật:

Giao Tiếp Đồng Bộ (REST)

Dùng cho các tác vụ cần phản hồi ngay lập tức (Login, Get Menu, Get Order Details).

Giao Tiếp Bất Đồng Bộ (Event-Driven)

Dùng RabbitMQ để xử lý tác vụ nền: Trừ kho, Gửi Email, Update Dashboard (Non-blocking I/O).

2. Xác Thực & Người Dùng

A. Đăng Ký (UC_DangKy)

Hệ thống kiểm tra trùng lặp Email trước khi tạo tài khoản. Mật khẩu được mã hóa bằng BCrypt.

sequenceDiagram
    actor U as Người dùng
    participant GW as API Gateway
    participant Auth as Service Auth
    participant DB as Auth DB

    U->>GW: POST /api/auth/register
    GW->>Auth: Forward Request
    Auth->>DB: Check Email Exists?
    alt Email Exists
        DB-->>Auth: True
        Auth-->>U: 409 Conflict
    else Email Valid
        Auth->>Auth: Hash Password
        Auth->>DB: Save User
        Auth-->>U: 201 Created
    end

B. Đăng Nhập (UC_DangNhap)

Sử dụng cơ chế JWT (JSON Web Token). Client lưu token và gửi kèm trong header của mọi request sau đó.

sequenceDiagram
    actor U as Người dùng
    participant Auth as Service Auth
    participant DB as Auth DB

    U->>Auth: POST /api/auth/login
    Auth->>DB: Find User
    DB-->>Auth: User Data
    Auth->>Auth: Verify Password
    alt Valid
        Auth->>Auth: Generate JWT Token
        Auth-->>U: 200 OK + Token
    else Invalid
        Auth-->>U: 401 Unauthorized
    end

3. Xem Thực Đơn & Tìm Kiếm

UC_XemMenu: Client load song song danh mục và danh sách món ăn để tối ưu tốc độ hiển thị.

sequenceDiagram
    actor U as Khách hàng
    participant GW as API Gateway
    participant Menu as Service Menu
    participant DB as Menu DB

    par Load Categories
        U->>GW: GET /api/categories
        GW->>Menu: Forward
        Menu->>DB: Select All Categories
        DB-->>Menu: List Categories
        Menu-->>U: JSON Categories
    and Load Products
        U->>GW: GET /api/menu?page=0
        GW->>Menu: Forward
        Menu->>DB: Select Products (Paged)
        DB-->>Menu: List Products
        Menu-->>U: JSON Products
    end

    opt Search
        U->>GW: GET /api/menu/search?q=Pizza
        GW->>Menu: Search
        Menu->>DB: ILIKE %Pizza%
        Menu-->>U: Search Results
    end

4. Quy Trình Đặt Hàng (Event-Driven)

Đây là quy trình phức tạp nhất. Service Order chỉ lưu trạng thái PENDING và trả về ngay để khách không phải chờ. Các việc nặng (Trừ kho, Gửi mail) chạy ngầm qua RabbitMQ.

sequenceDiagram
    autonumber
    actor U as Khách hàng
    participant Order as Order Svc
    participant MQ as RabbitMQ
    participant Inv as Inventory Svc
    participant Noti as Notification Svc

    U->>Order: POST /api/orders (Checkout)
    Order->>Order: Validate & Save (PENDING)
    Order->>MQ: Publish "order.created"
    Order-->>U: 201 Created
    
    par Inventory Process (Async)
        MQ->>Inv: Consume Event
        Inv->>Inv: Check & Deduct Stock
        alt Success
            Inv->>MQ: Publish "stock.success"
        else Out of Stock
            Inv->>MQ: Publish "stock.failed"
            MQ->>Order: Update Order -> CANCELLED
            Order-->>Noti: Trigger "Order Failed" Email
        end
    and Notification (Async)
        MQ->>Noti: Consume Event
        Noti->>U: Send Email "Order Confirmation"
    end

5. Thanh Toán Trực Tuyến

Tích hợp cổng thanh toán (SePay/VNPay mock). Sử dụng cơ chế Webhook (IPN) để cập nhật trạng thái đơn hàng một cách an toàn.

sequenceDiagram
    actor U as Khách hàng
    participant Pay as Payment Svc
    participant TP as Payment Gateway
    participant Order as Order Svc

    U->>Pay: Request Pay URL
    Pay->>TP: Create Payment Link
    TP-->>U: Redirect to Bank App

    U->>TP: Transfer Money
    
    par Webhook (Server-to-Server)
        TP->>Pay: Notify: Payment Success
        Pay->>Pay: Validate Transaction
        Pay->>Order: Calls API: Update Status -> PAID
        Order-->>Pay: OK
    end

6. Xử Lý Đơn Hàng (Staff)

UC_QuanLyDon: Khi nhân viên cập nhật trạng thái đơn (ví dụ: Đang giao), hệ thống sẽ gửi thông báo Realtime cho khách.

sequenceDiagram
    actor S as Nhân Viên
    participant Admin as Admin Dashboard
    participant Order as Service Order
    participant MQ as RabbitMQ
    actor C as Khách hàng

    S->>Admin: Click "Giao Hàng"
    Admin->>Order: PATCH /api/orders/{id}/status (DELIVERING)
    Order->>Order: Update DB
    Order->>MQ: Publish "order.status_change"
    Order-->>Admin: Success
    
    MQ->>C: Push Notification: "Đơn đang giao!"
